# MonoList v0.4.4 QA 记录

> 测试日期：2026-07-06

## 根因与修复

- 菜单栏入口由独立辅助进程承载，主进程原先不知道“待办”的真实屏幕位置，因此 Dock 只能回退到右上角。辅助进程现在会同步状态项 frame，Dock 与菜单栏统一使用该锚点。
- 主进程的全局外部点击监听会先收到辅助进程的菜单栏点击，导致清单先关闭、随后又被切换通知重新打开。监听现在会排除状态项范围。
- 主列表过去只在内容超过最大高度后创建滚动容器；现在始终保留垂直滚动弹性，同时继续按内容控制窗口高度。
- 草稿换行未计入自然高度；现在使用实际字体和输入宽度测算额外行数。
- 轻提醒使用固定公式计算高度并强制填满；现在由 SwiftUI 内容的 fitting size 决定面板高度。

# MonoList「今日专注」产品与交互设计

> 状态：已实现，待真实使用验证
> 确认日期：2026-07-20
> 视觉基准：`docs/design/focus-view-reference.html`

## 1. 结论

MonoList 不增加与普通清单对等的“专注模式”开关，而是在同一套任务数据上增加一个临时的**今日专注视图**：用户从现有短期任务和长期任务中按顺序选出 1–3 件今天最重要的事，进入后界面只呈现“当前一件 + 随后两件”。

这个方案保留普通清单随时收集、整理任务的能力，同时把执行阶段的视觉选择压缩到最多三件。它不是新的任务分类，也不会改变原任务的分组、顺序、提醒或完成状态。

## 2. 目标与非目标

### 2.1 目标

- 打开菜单栏窗口时，用户能立刻知道现在该做什么。
- 通过“当前一件 + 随后两件”降低在大量待办中反复选择的成本。
- 允许随时添加新待办，但新任务不会自动打断当前专注顺序。
- 用不抢焦点、不过度制造焦虑的提醒，帮助用户重新回到当前任务。
- 以可撤销、低成本的方式验证“每天最多三件”是否真的提高执行感。

### 2.2 非目标

- 不把 MonoList 变成番茄钟、日历排程或项目管理软件。
- 不新增第三种任务分组，也不复制一套独立任务数据。
- 不强迫用户每天必须选择三件；1–3 件均可开始。
- 不用持续弹窗、倒计时或高频声音制造压力。
- 首版不在提醒浮窗中提供完成、稍后提醒等复杂操作。

## 3. 产品模型

### 3.1 为什么不用对等模式切换

“普通 / 专注”分段开关会让用户误以为两个模式拥有不同数据或不同操作规则，也会产生“我现在到底在哪个模式”的记忆负担。

本设计采用有方向的路径：

```text
普通清单 → 选择今日专注 → 今日专注视图
    ↑                         ↓
    └──────── 查看全部 ───────┘
```

普通清单负责收集与整理，今日专注负责执行。两者是同一任务集合的不同视图，不是互斥工作模式。

### 3.2 核心状态

- **尚未设置**：普通清单显示“选择今日专注”。
- **正在选择**：在原列表内为任务标记 `①②③`，底部显示开始操作。
- **专注进行中**：只显示当天选定的 1–3 件，第一件为当前任务。
- **今日已完成**：仍显示原三件及完成状态，停止专注轻提醒。
- **新的一天**：不自动沿用昨天选择，回到选择入口；昨天未完成项作为候选建议。

## 4. 入口与任务选择

### 4.1 普通清单入口

- 未设置今日专注时，普通清单顶部出现文字按钮“选择今日专注”。
- 已设置但当前在普通清单时，按钮改为“继续专注”。
- 点击“继续专注”直接返回原今日专注视图，不重新进入选择流程，也不重置完成状态。
- 普通清单使用 `①②③` 标出当天已选任务，让用户在查看全部时仍能识别专注范围。
- 只要当天专注仍有效，菜单栏继续显示专注剩余数；查看全部或进入调整不会退回普通待办计数。
- 入口是上下文动作，不与“今天”标题组成分段控制器。

### 4.2 原位选择

- 点击“选择今日专注”后，直接在原清单进入选择状态，不弹出模态窗口。
- 短期任务和长期任务都可选择，已完成任务不可选择。
- 点击任务行为其依次分配 `①`、`②`、`③`；再次点击取消。
- 已选任务支持拖动调整顺序，序号随顺序更新。
- 选满三件后，其余任务保持可见但不可继续选择，并显示轻量说明“今日专注最多 3 件”。
- 选择 1 件后“开始专注”即可用，不要求凑满三件。
- “取消”退出选择状态，不改变之前已经生效的今日专注。

### 4.3 已设置后的调整

- 专注视图顶部提供“调整”。
- 点击“调整”后，选择页标题改为“调整今日专注”，主按钮改为“完成调整”；保存后直接返回专注视图。
- 调整过程中点击“取消”，恢复进入调整前的任务与顺序，并直接返回专注视图。
- 未完成任务可自由替换或重排，不弹确认框。
- 被移出的任务只退出今日专注，仍保留在原来的短期或长期分组和原位置。
- 已完成的专注任务当天锁定在本日记录中，不允许移出后补入第四件，避免“三件必办”退化为无限轮换。
- 删除任务时，该任务从今日专注中消失并留下空位；用户可主动补选，但删除记录不再展示。

## 5. 今日专注视图

### 5.1 窗口与层级

- 窗口宽度继续使用现有 `336 pt`，不改变菜单栏锚点。
- 常规高度约 `330–360 pt`；内容较长时最多增长到现有上限 `447 pt`。
- 三件任务占据主要内容区，不显示短期任务、长期任务、已完成分组等普通清单结构。
- 当前任务约占内容区的 45%–50%，使用 18 pt 半粗体，最多三行。
- 随后任务使用 14–15 pt，最多两行，视觉权重明显低于当前任务。
- 三件任务不做三个等权的大卡片；使用留白、字号和前后顺序建立层级。

### 5.2 顶部操作

顶部从左到右为：

- `今日专注` 与进度，如 `1/3`。
- 可拖动窗口的空白区域。
- “调整”。
- “全部”，返回普通清单。
- “＋”，随时新增待办。

“全部”表达查看完整任务集合，不使用容易产生模式误解的“退出专注”。它只切换视图，不结束今日专注；返回普通清单后，通过“继续专注”一键回到原专注页。

### 5.3 完成流转

- 点击当前任务圆圈后，当前任务收紧为一行完成态并保留在原位置。
- 下一件未完成任务上移成为当前任务，获得大字号和主要留白。
- 如果第二件先被完成，第一件仍是当前任务；完成的第二件缩为完成态，不改变未完成任务顺序。
- 全部完成后，三件任务均保留并显示勾选，顶部文案改为“今日完成”，不立即跳回普通清单。
- 当天取消完成时，该任务恢复到原专注顺序中；若它位于最前，则重新成为当前任务。

### 5.4 快速收集

- 专注视图始终保留“＋”。
- 新任务默认加入短期任务，不自动加入今日专注，不改变当前任务。
- 保存后显示短暂反馈：`已加入短期任务　调整今日专注`。
- 反馈中的“调整今日专注”可点击，进入原位调整状态。

## 6. 提醒系统

### 6.1 三层提醒

| 层级 | 作用 | 表现 |
|---|---|---|
| 菜单栏常驻状态 | 持续提供方向感 | 专注中显示剩余专注任务数；悬停提示当前任务；全部完成显示勾选状态 |
| 专注轻提醒 | 把用户温和拉回当前任务 | 只显示当前任务和进度，6 秒后收回，不抢焦点 |
| 单任务定时提醒 | 响应用户明确设定的时间承诺 | 优先级更高，可播放声音；点击后打开对应任务或专注视图 |

### 6.2 专注轻提醒内容

- 标题：`当前专注 · 1/3`。
- 正文只显示当前任务，不同时列出三件。
- 不显示“还有 N 秒关闭”的文字倒计时。
- 默认展示 6 秒；鼠标悬停暂停收回，移开后继续。
- 浮窗内不显示文字倒计时；6 秒时长只属于交互规则，不成为用户需要关注的信息。
- 点击提醒打开今日专注视图。
- 手动关闭只关闭本次提醒，下次仍按完整间隔出现。
- 使用非激活面板，不抢键盘焦点，不把当前应用切到 MonoList。

### 6.3 调度规则

- 沿用现有 30 / 60 / 90 / 120 分钟设置，推荐值为 60 分钟；不覆盖老用户已有选择。
- 以下行为视为用户已重新看见目标，并从此刻重新计算间隔：进入专注、打开专注窗口、完成任务、编辑或调整专注、切换当前任务。
- 主窗口打开时不展示专注轻提醒。
- Mac 唤醒后重新开始完整间隔，不立即弹出补偿提醒。
- 单任务定时提醒到达时，优先展示单任务提醒，并重置专注轻提醒计时。
- 超出用户设置的提醒时段、没有今日专注、当前任务为空或今日全部完成时，停止专注轻提醒。
- 设置页沿用现有“轻提醒”，增加说明：`设置今日专注后，轻提醒只提示当前任务。`

### 6.4 声音

- 专注轻提醒默认沿用现有“提醒声音”设置，但产品推荐关闭声音。
- 单任务定时提醒可以按现有声音设置播放一次提示音。
- 不新增音量、音色或每层独立声音配置。

## 7. 日期边界

- “一天”按本地时区的凌晨 `04:00` 切换，避免午夜仍在工作时突然清空专注视图。
- 到新的一天后，不自动把昨天的选择设为今天专注。
- 昨天未完成的专注任务在选择状态顶部作为“昨天未完成”候选展示，用户确认后才进入今天。
- 文案保持中性，不使用“逾期”“失败”“未完成目标”等压力表达。

## 8. 数据与状态约束

今日专注是独立的视图状态，建议存储为：

```swift
struct DailyFocusSelection: Codable {
    var dayKey: String        // 以本地 04:00 为日界线生成
    var orderedTaskIDs: [UUID] // 1...3，顺序即优先级
}
```

- 任务正文、分组、排序、完成时间和单任务提醒继续由 `TaskItem` 维护。
- 今日专注只保存任务 ID 与顺序，不复制任务内容。
- 加载时过滤已删除 ID，但保留仍存在的已完成任务。
- 数据写入使用现有原子写入和恢复机制；若专注记录损坏，只丢弃该视图状态，不能影响任务主数据。

## 9. 键盘、可访问性与动效

- 选择状态支持方向键移动、空格选中或取消、`Command + Return` 开始专注、`Escape` 取消。
- 专注视图支持方向键移动当前选择、空格完成或取消完成；输入状态仍遵循现有 Enter 规则。
- 序号同时用文字 `1/2/3` 提供辅助功能标签，不能只靠颜色区分。
- 所有顶部按钮保持至少 28 × 28 pt 命中区域。
- 视图切换约 200 ms；完成后主次任务交接约 220 ms；减少动态效果开启时取消位移，仅保留即时状态变化或轻微淡化。

## 10. 实验方式

首版作为实验功能发布，但不增加远程行为分析：

- 以 7–14 天自用与小范围反馈验证。
- 重点观察：是否每天愿意设置、是否更快回到当前任务、提醒是否被频繁直接关闭、是否经常替换未完成任务、三件上限是否足够。
- 成功信号不是“完成率绝对提高”，而是用户打开 MonoList 后更少重新浏览整张清单，并能明确说出当前任务。
- 若多数使用仍频繁返回全部清单或每天需要超过三件，优先调整选择与展示规则，而不是增加更强提醒。

## 11. 首版范围

### 必须实现

- 1–3 件原位选择、排序与调整。
- 今日专注视图及当前任务交接。
- 专注中快速新增但不自动插队。
- 菜单栏专注剩余数与当前任务提示。
- 专注轻提醒内容、调度重置和优先级规则。
- 04:00 日界线与昨天未完成候选。

### 暂不实现

- 番茄钟、计时统计、连续天数或成就。
- AI 自动挑选三件事。
- 提醒浮窗内直接完成或稍后提醒。
- 为不同专注任务设置独立颜色、图标或背景。
- 远程数据分析、账户同步或跨设备同步。

## 12. 验收标准

1. 普通清单不出现对等的“普通 / 专注”模式开关。
2. 用户可以从短期和长期任务中选择 1–3 件，并明确看到顺序。
3. 专注视图默认只呈现选定任务，第一件视觉权重明显高于随后任务。
4. 新增任务不会自动进入专注或改变当前任务。
5. 完成当前任务后，下一件平滑成为当前任务；已完成任务当天仍保留。
6. 当天不能通过移除已完成任务补入第四件。
7. 专注轻提醒只显示当前任务，6 秒自动收回，悬停暂停且不抢焦点。
8. 主窗口打开、超出提醒时段、Mac 刚唤醒或今日全部完成时不会错误弹出专注轻提醒。
9. 单任务定时提醒与专注轻提醒同时到期时，只展示单任务提醒。
10. 本地时间跨过 04:00 后要求重新选择，昨天未完成任务只作为建议。
11. 今日专注状态损坏或引用任务被删除时，普通任务数据仍可正常加载。
12. 键盘操作、辅助功能标签和“减少动态效果”均可正常工作。

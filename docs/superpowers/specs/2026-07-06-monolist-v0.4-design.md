# MonoList v0.4 交互稳定性与轻提醒设计

## 目标

本次迭代集中解决清单窗口弹跳、编辑与新增交互不一致、图标复杂、按钮反馈不足，以及轻提醒动效不完整的问题。实现以稳定、简洁、符合 macOS 使用习惯为优先，不增加无关功能。

## 一、应用图标

- Dock 图标和控制台 Logo 统一改为白色圆角底、黑色图形。
- 图形只保留一个圆圈和一个对号，不再展示多行清单或横线。
- 圆圈与对号使用清晰、克制的线宽，保证小尺寸下仍可辨认。
- 图标四周保留符合 macOS Dock 图标比例的安全边距。

## 二、清单窗口

### 2.1 尺寸变化

- 窗口顶部位置在显示期间保持不变。
- 新增、删除、完成任务，以及显示或隐藏历史记录时，只允许下边缘向下展开或向上收起。
- 窗口尺寸只由 `WindowCoordinator` 控制，SwiftUI 内容不再与窗口动画争夺尺寸。
- 连续发生的高度变化合并处理，并取消尚未完成的旧尺寸动画，避免窗口闪动和弹跳。
- Dock 和菜单栏唤起窗口时保留淡入与轻微展开动效，节奏比当前版本稍慢，但不影响操作效率。

### 2.2 布局和背景

- 清单窗口使用纯白色背景。
- “已完成”区域与窗口下边缘之间始终保留固定空隙。
- 底部空隙不随已完成任务数量变化。
- 内容未达到最大高度时不显示滚动；达到上限后才启用内部滚动。

### 2.3 顶部操作按钮

- 加号、更多操作和设置按钮统一为灰色圆角方形背景。
- 三个按钮使用相同尺寸、圆角、默认颜色、悬停反馈和按压反馈。
- 更多操作按钮不再出现缺失背景的情况。

## 三、待办编辑与新增

### 3.1 编辑已有待办

- 双击已有待办进入编辑状态。
- 按 Enter 保存。
- 点击清单空白处、表头、窗口边缘或其他按钮时，先保存当前编辑内容，再执行点击对应的操作。
- 如果编辑后内容为空，恢复进入编辑前的原内容，不删除任务。

### 3.2 新增待办

- 点击加号后，在未完成任务末尾显示输入行并自动聚焦。
- 按 Enter 保存。
- 点击清单空白处、表头、窗口边缘或其他按钮时，也保存当前输入，再执行点击对应的操作。
- 如果输入内容为空，点击其他位置时取消新增，不创建空任务。

### 3.3 焦点协调

- 编辑和新增共用明确的“提交当前输入”入口。
- 外部点击先提交输入，再清除焦点，避免焦点变化和按钮操作分别触发两次界面更新。
- 删除、完成和显示历史记录等操作不会让窗口重新获取错误焦点。

## 四、控制台交互

- 提醒间隔、提醒位置和立即测试三个右侧控件保持统一宽度和字号。
- 下拉选择控件不显示箭头。
- 控件默认使用安静的灰色背景；悬停和按下时提高背景对比度。
- “立即测试”第一次点击显示完整轻提醒，第二次点击关闭当前测试提醒。
- 如果测试提醒已自动关闭，按钮恢复为可再次测试的状态。

## 五、轻提醒

### 5.1 展开和收回

- 顶部居中提醒从顶部锚点以小圆形出现，再向下展开为完整矩形。
- 右上角提醒使用右上方锚点执行同样的展开过程。
- 收回时先从矩形缩小为圆形，再平滑缩回对应的屏幕顶部锚点。
- 内容在展开过程中渐显，在收回过程中渐隐。
- 动效节奏比当前版本稍慢，保持清楚但不拖沓。

### 5.2 音效

- 每次真正展示轻提醒时播放一次轻柔的 macOS 系统提示音。
- “立即测试”执行与自动提醒完全相同的展示、动画和音效。
- 关闭提醒不播放音效。

## 六、实现边界

- 不增加提醒音效选择、音量设置或自定义音频文件。
- 不增加新的提醒位置。
- 不改变任务数据格式和历史记录规则。
- 不改变自动更新、签名和发布流程。
- 本轮先完成本地构建和验证；是否发布新版本由后续指令决定。

## 七、验收标准

1. 连续新增、删除、完成任务和切换历史记录时，窗口顶部位置误差不超过 1 点。
2. 窗口高度变化只发生在下边缘，不出现先上跳再拉长的过程。
3. 已有待办和新增待办在按 Enter 或点击任意空白区域时都能正确保存。
4. 空白新增内容不会产生任务。
5. 清单背景为纯白，底部空隙在零条、一条和多条已完成任务时保持一致。
6. 三个顶部按钮外观和点击反馈一致。
7. 控制台下拉控件无箭头，三个右侧控件宽度和字号一致。
8. “立即测试”可通过第二次点击关闭提醒。
9. 自动提醒和测试提醒均包含完整展开、收回动效及一次提示音。
10. Dock 图标和控制台 Logo 均显示简化后的圆圈对号图形。

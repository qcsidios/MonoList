# MonoList v0.4.2 稳定性修复设计

## 目标

修复 v0.4.1 菜单栏状态项失效造成的窗口错误定位，并完成紧凑布局、空白双击新增、编辑后连续输入、设置控件点击和构建产物管理优化。

## 核心规则

- 菜单栏恢复原生文字状态项，不再使用 `NSTextAttachment`。
- 无任务显示“待办”；有任务显示“待办 N”，数字使用橘色强调。
- 菜单栏按钮无法提供窗口时，锚点回退到主屏幕顶部右侧，不使用鼠标位置。
- 清单默认紧贴菜单栏，不保留额外垂直间隔。
- 清单仍可拖动，动态高度变化保持用户当前顶部位置。
- 清单显示使用轻微下滑和淡入，关闭使用上收和淡出。

## 交互

- 双击清单空白处创建新的末尾输入行。
- 编辑已有待办时，点击空白处保存并结束。
- 编辑已有待办时按 Enter，先保存，再在该任务下方创建输入行。
- 新增输入仍支持 Enter 保存并继续下一行。

## 布局

- 已完成任务最后一行到窗口底边使用与列表顶部相近的紧凑间距。
- 轻提醒高度按实际内容计算，减少上下留白。
- 设置选择控件的文字区域和右侧箭头区域均可弹出菜单。

## 更新与构建

- 更新状态明确显示检测中、发现版本、已是最新版、下载中和安装中。
- 构建产物固定为 `build/local/MonoList.app`。
- 发布包固定为 `build/release/MonoList-v0.4.2.dmg`。
- 发布完成后删除 `build/tests` 和 `build/release/dmg-staging`，保留最新本地 App 与 DMG。
- 清理 `.DS_Store` 和临时预览，不删除源码、测试、发布说明与已确认的 UI 设计稿。

## 验收

1. 菜单栏状态项具有有效屏幕坐标，点击后清单位于菜单栏下方。
2. Dock 点击在状态栏按钮异常时仍从主屏幕顶部打开。
3. 清单顶部紧贴菜单栏并可自由拖动。
4. 双击空白处可新增；编辑 Enter 可保存并在下方新增。
5. 列表底部与提醒框留白紧凑。
6. 下拉箭头区域可正常点击。
7. `build/local` 与 `build/release` 保留最新产物，临时目录自动清理。
8. v0.4.2 通过应用内更新发布。

# MonoList v0.4 实施计划

> **执行要求：** 使用测试驱动开发逐项实施。每个行为先补失败测试，再写最小实现并验证。

**目标：** 修复清单窗口弹跳和输入提交问题，统一界面反馈，并完成简化 Logo 与完整轻提醒体验。

**架构：** 继续使用现有 AppKit 窗口协调器承载 SwiftUI 界面。窗口尺寸、提醒面板尺寸和动画锚点由各自控制器单点管理；SwiftUI 只报告目标内容高度和展示状态。编辑、新增和提醒测试状态分别收敛到现有状态对象，不引入新的业务层。

**技术栈：** Swift、SwiftUI、AppKit、现有 Shell Smoke Tests。

---

## 文件职责

- `MonoList/App/WindowCoordinator.swift`：主清单窗口显示、顶部锚定与尺寸动画。
- `MonoList/Tasks/TaskListView.swift`：清单布局、空白点击、草稿提交和顶部按钮。
- `MonoList/Tasks/TaskDraftState.swift`：新增待办的提交与空内容取消规则。
- `MonoList/Tasks/TaskRowView.swift`：已有待办失焦自动保存。
- `MonoList/Settings/SettingsView.swift`：控制台控件反馈和测试提醒开关。
- `MonoList/Reminder/ReminderPanelController.swift`：提醒状态、位置、展开收回动画和音效。
- `MonoList/Reminder/ReminderView.swift`：提醒内容渐显及圆形阶段外观。
- `MonoList/App/AppDelegate.swift`：测试提醒开关接线。
- `MonoList/Shared/MonoListLogoView.swift`：控制台简化 Logo。
- `scripts/generate-app-icon.swift`：生成与控制台一致的 Dock 图标。
- `Tests/WindowCoordinatorSmoke.swift`：窗口锚定和草稿提交回归测试。
- `Tests/ReminderSchedulerSmoke.swift`：提醒锚点、测试状态和音效触发测试。
- `Tests/AppLaunchSmoke.swift`：图标安全边距和视觉结构检查。

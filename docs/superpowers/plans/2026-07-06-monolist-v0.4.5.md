# MonoList v0.4.5 清单高度与输入交互修复计划

**目标：** 让清单窗口始终贴合实际内容；以“七条两行待办”为最大可见高度，超过后内部滚动；统一新增、编辑、保存和圆圈对齐。

**实现约束：**

- 一条两行待办不能被高度算法误算成三行。
- 最大窗口完整显示七条两行待办及“已完成”表头，不保留额外底部空白。
- 第八条开始通过窗口内部滚动查看。
- Enter 保存当前内容并立即进入下一条空白输入；点击空白处保存并结束输入。
- 加号和双击空白处新增待办；双击已有待办进入编辑。
- 空白输入行不显示提示文案。
- 一行和两行待办的左侧圆圈均相对整行垂直居中。
- 新建设置的提醒间隔默认为 30 分钟；已有用户设置保持不变。
- 控制台下拉文字相对整个控件居中，箭头贴紧最右侧。
- 发布后仓库本地只保留最新版 DMG，不保留构建 App、旧 DMG 或测试产物。

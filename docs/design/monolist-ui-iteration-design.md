# MonoList 本地迭代 UI 设计基准

> 状态：持续补充  
> 首次确认日期：2026-07-05  
> 当前确认范围：今天视图、已完成任务与历史记录的展示逻辑  
> 对应代码稿：`docs/design/completed-tasks-reference.html`

## 1. 文档用途

本文件和对应 HTML 是本轮 MonoList 本地迭代的视觉与交互基准。开发时必须直接对照代码稿实现，不得仅凭聊天记录或记忆重新设计。

如果实现效果与代码稿不一致，以代码稿和本文件中已经确认的规则为准。需要改变已确认设计时，必须先向青城展示新方案并获得确认。

## 2. 产品参照

待办清单的主要交互逻辑优先参考 macOS 系统自带的“提醒事项”，包括：

- 在当前列表中直接新增任务。
- 使用圆形勾选控件表示完成状态。
- 已完成任务可以在当前列表中显示或隐藏。
- 完成记录按日期组织。
- 不为了查看完成任务强制切换到独立历史页面。

开源项目只用于补充实现思路。若开源项目逻辑与 Apple“提醒事项”冲突，优先采用 Apple 的管理方式。

## 3. 今天与全部

主浮窗提供“今天 / 全部”两个视图：

- “今天”展示今天的任务和以前尚未完成的任务。
- 以前尚未完成的任务继续位于“今天”顶部，不显示“逾期”标签。
- “全部”用于查看所有未完成任务，不改变任务原本的日期属性。
- 今天创建的任务默认属于今天。

## 4. 完成任务流转

1. 用户点击任务前方圆圈后，任务从未完成区域移动到下方“已完成”区域。
2. 今天完成的任务在当天始终展示在“已完成”区域。
3. 到第二天后，这些任务不再作为“今天完成”展示，但继续永久保留在历史记录中。
4. 用户取消完成时，任务恢复为未完成，并回到未完成任务区域。

## 5. 已完成区域

已完成区域位于未完成任务下方，标题行右侧靠近边框放置“显示 / 隐藏”按钮。

### 隐藏状态

- 今天完成的任务继续正常展示。
- 以前日期的历史完成记录完全隐藏。
- 按钮文字为“显示”。

### 显示状态

- 今天完成的任务位于最前。
- 更早的完成记录继续向下展开。
- 历史记录按完成日期分组，并显示日期标题。
- 按钮文字为“隐藏”。
- 内容超过浮窗最大高度后，只让列表区域滚动。

## 6. 取消独立历史页面

删除当前单独的历史记录视图、历史记录入口和返回按钮。

恢复任务、查看完成日期、删除单条完成记录和清空完成记录等操作，都在主浮窗的“已完成”区域中完成。

## 7. 当前视觉基准

当前确认的代码稿使用以下视觉规则：

- 浮窗采用浅色半透明玻璃材质。
- 外层使用连续圆角和轻量阴影。
- 未完成任务使用圆形空心勾选区域。
- 完成任务使用对勾、次级文字颜色和删除线。
- “显示 / 隐藏”使用尺寸足够、状态明确的胶囊按钮。
- 日期标题使用较小的次级文字，与任务内容形成清晰层级。
- 按钮命中区域不得只等于图标本身，最小点击区域按 28 × 28 pt 设计。

具体尺寸、颜色和排列以 `completed-tasks-reference.html` 为准。

## 8. 一致性检查

开发完成后至少进行以下对照：

- 截取实际“隐藏历史”状态，与代码稿左侧比较。
- 截取实际“显示历史”状态，与代码稿右侧比较。
- 核对今天完成项在两种状态下都不会消失。
- 核对历史记录只在“显示”状态出现。
- 核对完成记录日期分组和顺序。
- 核对窗口内容较少时不会保持过高空白区域。

## 9. 后续补充方式

Logo、设置窗口、空列表输入状态、任务编辑状态和动态展开效果确认后，继续在本目录增加对应 HTML 代码稿，并在本文件追加章节。所有已确认代码稿进入 Git 版本管理。

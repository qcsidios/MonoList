# MonoList 本地迭代 UI 设计基准

> 状态：持续补充  
> 首次确认日期：2026-07-05  
> 当前确认范围：今天视图、新增草稿、已完成任务与历史记录的展示逻辑
> 对应代码稿：`docs/design/completed-tasks-reference.html`、`docs/design/task-input-reference.html`

## 1. 文档用途

本文件和对应 HTML 是本轮 MonoList 本地迭代的视觉与交互基准。开发时必须直接对照代码稿实现，不得仅凭聊天记录或记忆重新设计。

如果实现效果与代码稿不一致，以代码稿和本文件中已经确认的规则为准。需要改变已确认设计时，必须先向青城展示新方案并获得确认。

## 2. 产品参照

待办清单的主要交互逻辑优先参考 macOS 系统自带的“提醒事项”，包括：

- 在当前列表中直接新增任务。
- 使用圆形勾选控件表示完成状态。
- 已完成任务可以在当前列表中显示或隐藏。
- 完成记录按日期组织。
- 不为了查看完成任务强制切换到独立历史页面。

开源项目只用于补充实现思路。若开源项目逻辑与 Apple“提醒事项”冲突，优先采用 Apple 的管理方式。

## 3. 单一“今天”视图

主浮窗只保留“今天”一个主视图，不提供“今天 / 全部”切换：

- 所有新任务都默认属于今天。
- 以前尚未完成的任务继续位于“今天”顶部，不显示“逾期”标签。
- 未完成任务会一直留在“今天”，直到用户完成或删除。
- 不增加与“今天”内容重复的“全部”视图。
- 原加号按钮不再作为新增入口；新增入口直接融入列表。

## 4. 完成任务流转

1. 用户点击任务前方圆圈后，任务从未完成区域移动到下方“已完成”区域。
2. 今天完成的任务在当天始终展示在“已完成”区域。
3. 到第二天后，这些任务不再作为“今天完成”展示，但继续永久保留在历史记录中。
4. 用户取消完成时，任务恢复为未完成，并回到未完成任务区域。

## 5. 新任务草稿

新增入口采用列表内联输入，不使用独立加号或底部胶囊输入栏：

- 输入行本身就是列表中的一行。
- 无论输入框是否为空，左侧始终显示圆形勾选区域。
- 空列表时只展示标题和一条输入行，不保留大面积空白。
- 已有任务时，输入行与普通任务保持相同宽度、圆角和水平对齐。
- 双击列表空白处，在点击位置生成并聚焦同样的输入行。
- 任一时刻最多存在一个新任务草稿。

新增任务只有在用户按下 `Enter` 后才成为正式任务并写入任务数据。

用户仍在输入、停顿思考或尚未按下 `Enter` 时：

- 不执行自动保存。
- 不创建任务 ID。
- 不计入菜单栏数量。
- 不进入提醒内容。
- 关闭浮窗后，在本次应用运行期间保留内存草稿。
- 重新打开浮窗时恢复草稿和输入位置，允许继续编辑。
- 正常退出应用后丢弃该草稿，不写入磁盘。

### 浮窗动态尺寸

- 浮窗宽度固定为 360 pt。
- 空列表最小状态只容纳标题区和输入行，参考高度为 148 pt。
- 新任务保存后，浮窗以平滑动画向下扩展。
- 内容较少时，浮窗高度按实际任务行数增长。
- 达到最大高度后停止增长，只让列表内容滚动。
- 尺寸动画参考 `task-input-reference.html`，使用约 220–280 ms 的平滑缓动，不使用弹跳幅度过大的动画。

## 6. 已完成区域

已完成区域位于未完成任务下方，标题行右侧靠近边框放置“显示 / 隐藏”按钮。

### 隐藏状态

- 今天完成的任务继续正常展示。
- 以前日期的历史完成记录完全隐藏。
- 按钮文字为“显示”。

### 显示状态

- 今天完成的任务位于最前。
- 更早的完成记录继续向下展开。
- 历史记录按完成日期分组，并显示日期标题。
- 按钮文字为“隐藏”。
- 内容超过浮窗最大高度后，只让列表区域滚动。

## 7. 取消独立历史页面

删除当前单独的历史记录视图、历史记录入口和返回按钮。

恢复任务、查看完成日期、删除单条完成记录和清空完成记录等操作，都在主浮窗的“已完成”区域中完成。

## 8. 当前视觉基准

当前确认的代码稿使用以下视觉规则：

- 浮窗采用浅色半透明玻璃材质。
- 外层使用连续圆角和轻量阴影。
- 未完成任务使用圆形空心勾选区域。
- 完成任务使用对勾、次级文字颜色和删除线。
- “显示 / 隐藏”使用尺寸足够、状态明确的胶囊按钮。
- 日期标题使用较小的次级文字，与任务内容形成清晰层级。
- 按钮命中区域不得只等于图标本身，最小点击区域按 28 × 28 pt 设计。

具体尺寸、颜色和排列以 `completed-tasks-reference.html` 和 `task-input-reference.html` 为准。

## 9. 一致性检查

开发完成后至少进行以下对照：

- 截取实际“隐藏历史”状态，与代码稿左侧比较。
- 截取实际“显示历史”状态，与代码稿右侧比较。
- 核对今天完成项在两种状态下都不会消失。
- 核对历史记录只在“显示”状态出现。
- 核对完成记录日期分组和顺序。
- 核对窗口内容较少时不会保持过高空白区域。
- 核对未按 `Enter` 的草稿不会创建任务，关闭并重新打开浮窗后仍可继续输入。
- 核对退出应用再启动后，未提交草稿已经消失。

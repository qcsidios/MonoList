# MonoList v0.4.3 QA 记录

> 测试日期：2026-07-06

## 本轮修复

- 已确认根因：macOS 26 将旧 Bundle ID 的状态项保留在不可见位置；即使系统设置已允许 MonoList 显示，直接重建状态项也无法恢复。
- 菜单栏入口改由独立轻量组件承载，仍显示“待办 N”，主应用 Bundle ID、签名、数据目录和权限身份均保持不变。
- 主清单使用无标题栏面板，去除透明标题栏造成的视觉空隙。
- 菜单栏锚点和 Dock 回退锚点均直接对齐菜单栏下边缘。
- 窗口背景不再接受拖动，只在表头预留的空白区域执行窗口拖动。
- 面板会在鼠标点击当前文本编辑器以外的位置时结束编辑，由现有失焦逻辑完成保存。
- 设置页恢复白底和浅灰卡片；下拉选择保留原有灰度样式，并将文字与箭头合并为同一个完整点击区域。
- 更新按钮高度固定为 22 pt，更新状态和升级入口位于版本号右侧。
- 根据实际任务行数动态计算面板高度，最后一条任务与底边保持紧凑且稳定的间距。

## 实机检查

- 使用正式固定签名启动 v0.4.3，菜单栏正常显示“待办 2”。
- 点击菜单栏文字后，清单窗口显示于 `(794, 34)`，尺寸为 336 × 428 pt；顶部以系统可见工作区为基准，不遮挡菜单栏，也不保留额外空隙。
- 将待办数量通知更新为 5 后，菜单栏立即显示“待办 5”。
- 主应用退出后，菜单栏辅助组件同步退出，不残留测试进程。
- 设置页为白色背景和浅灰卡片；实测点击“提醒间隔”最右侧箭头区域可以正常展开菜单。
